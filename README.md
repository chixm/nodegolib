# nodegolib

GoでビルドしたDLL（共有ライブラリ）をNode.jsから簡単に呼び出すためのライブラリです。  
Windows上のElectronアプリケーションなど、Node.js環境からGoのロジックを利用したい場合に活用できます。

> **対象プラットフォーム: Windows のみ**

---

## アーキテクチャ概要

```
Node.js (index.js)
    │
    │  require('bindings')
    ▼
C++ Addon (addon.cc)  ←  node-addon-api / N-API
    │
    │  LoadLibrary / GetProcAddress (Windows API)
    ▼
Go DLL (include/goshared.dll)
    │
    │  cgo でエクスポートした関数
    ▼
Go ロジック (include/main.go)
```

Node.js → C++ Addon → Go DLL という3層構造で呼び出しが行われます。  
`addon.cc` / `index.js` / `index.d.ts` / `index.test.js` は **`scripts/generate.js` によって自動生成**されるため、手動での編集は不要です。

## セットアップ

### 1. 依存パッケージのインストール

```bash
npm install
```

### 2. フルビルド

```bash
npm run build
```

内部では以下の3ステップが順に実行されます。

| ステップ | コマンド | 内容 |
|---------|---------|------|
| 1 | `npm run build:dll` | Go ソースを DLL にコンパイル |
| 2 | `npm run generate` | `main.go` を解析してソースを自動生成 |
| 3 | `npm run build:addon` | C++ Addon をビルド |

## Go 関数を追加する手順

### 1. `include/main.go` に関数を書く

`//export 関数名` コメントを付けて関数を追加します。

```go
package main

import "C"

func main() {}

//export Greet
func Greet(name *C.char) *C.char {
    goName := C.GoString(name)
    return C.CString("Hello " + goName + " from Go!\n")
}

//export Add
func Add(a *C.char) *C.char {
    // 複数の関数も同様に追加できる
    return C.CString("result: " + C.GoString(a))
}
```

**ルール:**
- `import "C"` と空の `main()` が必須です
- エクスポートする関数には必ず `//export 関数名` を付けます
- 引数・戻り値は `*C.char`（文字列）のみサポートしています
- 引数は複数でも可です

### 2. ビルドを実行する

```bash
npm run build
```

これだけで以下が自動更新されます。

| ファイル | 生成内容 |
|---------|---------|
| `addon.cc` | 各 Go 関数に対応する C++ AsyncWorker と Node.js バインディング |
| `index.js` | Promise ベースの async ラッパー関数 |
| `index.d.ts` | TypeScript 型定義 |
| `index.test.js` | 各関数の動作確認テスト |

### 3. 関数を呼び出す

Go の関数名は **先頭小文字のキャメルケース** で JS に公開されます。

| Go 関数名 | JS 関数名 |
|----------|---------|
| `ExecuteAsync` | `executeAsync` |
| `Greet` | `greet` |
| `Add` | `add` |

```js
const { greet, add } = require('./index');

const result = await greet("World");
console.log(result); // => "Hello World from Go!"
```

## テスト

```bash
npm test
```

`index.test.js` は自動生成されており、全エクスポート関数に対して「文字列が返ること」を検証します。

```js
// 自動生成されるテストの例
const { greet } = require('./index');

describe('Go DLL exports', () => {
  test('greet resolves to a string', async () => {
    const result = await greet("test1");
    expect(typeof result).toBe('string');
    expect(result.length).toBeGreaterThan(0);
  }, 10000);
});
```

戻り値の具体的な内容を検証したい場合は、`index.test.js` を直接編集してください（次の `npm run build` で上書きされます）。継続的にカスタムテストを管理したい場合は別ファイル（例: `custom.test.js`）に書くことを推奨します。

## コードの自動生成について

`scripts/generate.js` が `include/main.go` を解析して各ファイルを生成します。  
`npm run generate` で単体実行も可能です。

```bash
npm run generate
# Found 2 exported function(s): ExecuteAsync, Greet
#   generated addon.cc
#   generated index.js
#   generated index.d.ts
#   generated index.test.js
```

生成されたファイルには `// AUTO-GENERATED by scripts/generate.js — do not edit manually` と記載されています。手動で編集しても次のビルド時に上書きされます。

## プロジェクト構成

```
nodegolib/
├── include/
│   ├── main.go          # Go の実装（ここだけ編集する）
│   ├── goshared.dll     # ビルド済み Go DLL（自動生成）
│   └── goshared.h       # cgo が自動生成するヘッダ
├── scripts/
│   └── generate.js      # コード自動生成スクリプト
├── addon.cc             # C++ Addon（自動生成）
├── binding.gyp          # node-gyp のビルド設定
├── index.js             # Node.js 公開 API（自動生成）
├── index.d.ts           # TypeScript 型定義（自動生成）
├── index.test.js        # Jest テスト（自動生成）
└── package.json
```

## ビルドフロー詳細

```
include/main.go
    │
    │  npm run build:dll
    │  go build -buildmode=c-shared
    ▼
include/goshared.dll  +  include/goshared.h
    │
    │  npm run generate
    │  scripts/generate.js が //export を解析
    ▼
addon.cc / index.js / index.d.ts / index.test.js（自動生成）
    │
    │  npm run build:addon
    │  node-gyp rebuild（binding.gyp でDLLを build/Release/ へコピー）
    ▼
build/Release/addon.node
    │
index.js  ──(require('bindings')('addon'))──▶  addon.node をロード
```
